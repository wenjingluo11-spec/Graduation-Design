## 根本原因

这两个错误都是由 **cuDNN 内部错误** 引起的，常见原因：

1. **GPU 内存碎片化** - 连续训练导致显存管理问题
2. **批次大小过大** - 超出 GPU 内存限制
3. **数值不稳定** - 梯度爆炸或 NaN 值
4. **cuDNN 版本问题** - PyTorch 与 cuDNN 不兼容

   - 显示 CUDA/cuDNN 版本信息

2. **NaN 检测** (Seq2Seq 和 Transformer)

### 方案 2: 减小批次大小 ⭐⭐⭐⭐

找到超参数配置单元格（第 3 个单元格），修改：

```python

```

**原理**：减少每批次的数据量，降低 GPU 内存压力

---

### 方案 3: 使用 CPU 训练 ⭐⭐⭐

找到超参数配置单元格，修改：

```python

```

**优点**：稳定，不会出错
**缺点**：速度慢（约慢 10-50 倍）

## 训练监控

训练时注意观察：

1. **Loss 值**：应该逐渐下降，不应该出现 NaN 或 Inf
2. **GPU 内存**：不应该接近 100%
3. **训练速度**：每个 epoch 应该在合理时间内完成

如果出现以下情况，立即停止：
- Loss 变成 NaN
- GPU 内存溢出
- 训练速度异常缓慢

## 常见问题 FAQ

### Q: 为什么 Seq2Seq 能训练，Transformer 不行？

A: Transformer 模型更大，需要更多 GPU 内存。解决方法：
- 减小 `BATCH_SIZE`
- 减小 `D_MODEL`（模型维度）
- 减小 `NUM_LAYERS`（层数）

### Q: 训练速度太慢怎么办？

A: 如果使用 CPU 训练：
- 减少 `EPOCHS`（训练轮数）
- 减少训练数据量
- 使用更小的模型

### Q: 如何判断 GPU 内存是否足够？

A: 运行 GPU 诊断单元格，查看：
```
GPU 内存: X.XX GB
```
如果小于 4GB，建议使用 CPU 或减小 BATCH_SIZE
